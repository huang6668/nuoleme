# 挪了么

挪了么是一个离线、轻量、零常驻的 Android 挪车短信提醒 App。它通过静态 `SmsReceiver` 监听 `SMS_RECEIVED`，命中挪车关键词或车牌后优先启动短生命周期 `AlarmService` 播放报警音、震动并展示前台通知；当 Android 12+ 后台前台服务启动受限时，会自动退化为高优先级/全屏报警通知，并由 `AlarmActivity` 作为锁屏提醒兜底。

## 隐私与权限

- 不联网，不申请 `INTERNET` 权限。
- 不上传短信，不集成统计、广告、崩溃上报 SDK。
- 不读取历史短信，不需要默认短信 App。
- 只申请运行所需权限：`RECEIVE_SMS`、`POST_NOTIFICATIONS`、`VIBRATE`、前台服务权限、全屏提醒权限。

## 功能概览

- 静态 `SmsReceiver` 监听来信，平时 0 常驻。
- 自定义关键词和车牌匹配，设置使用 `SharedPreferences` 保存。
- 报警支持时长、震动、系统默认闹铃铃声、测试报警。
- `AlarmActivity` 作为锁屏/全屏提醒兜底，并在最近任务中隐藏。
- GitHub Actions 自动构建 debug APK 并上传 artifact。

## 安装方式

1. 打开仓库的 GitHub Actions。
2. 运行或等待 `Android Build` 工作流完成。
3. 在构建结果中下载 `nuolemo-debug-apk` artifact。
4. 把 `app-debug.apk` 安装到手机。

## 首次使用

1. 打开 App，允许短信权限。
2. Android 13 及以上再允许通知权限。
3. 检查设置页中的全屏提醒状态。
4. 点击“测试报警”确认系统闹铃、震动和通知都正常。

## 全屏报警与保活建议

Android 14 及以上系统支持单独关闭全屏提醒，设置页提供了对应状态和入口。如果你希望在息屏、锁屏或厂商 ROM 较严格的环境里尽量减少漏报，建议同时完成这些系统设置：

- 允许应用自启动。
- 允许后台运行或后台活动。
- 电池优化改为“不受限”或“无限制”。
- 避免把 App 加入一键清理名单。

小米、华为、OPPO、vivo、三星等 ROM 的入口名称不同，但一般都能通过应用详情页、电池设置页或通知设置页完成上述配置。

## 测试方法

- 在设置页点击“测试报警”，验证系统闹铃、震动、前台通知和停止按钮。
- 用另一台手机发送包含“挪车”“移车”或你自定义关键词/车牌的短信，验证自动触发。
- 发送验证码类短信，确认不会误报。

## 常见问题

### 收不到提醒怎么办？

- 先确认短信权限、通知权限和全屏提醒状态都正常。
- 在系统设置里允许自启动、后台运行，并关闭电池优化。
- 确认短信文本真的包含挪车关键词或你配置的车牌。

### 为什么需要短信权限？

因为 App 依赖系统广播里的当前短信内容做本地匹配，不读取历史短信，也不上传任何数据。

### 为什么没有网络权限？

这是刻意的设计目标。App 全离线运行，不依赖服务端，也不会联网传输短信内容。

### 需要设置成默认短信 App 吗？

不需要。挪了么只监听系统下发的 `SMS_RECEIVED` 广播，不拦截、不接管短信应用。

### 为什么报警窗口不会出现在最近应用？

`AlarmActivity` 只作为临时锁屏/全屏提醒入口，Manifest 中设置了 `excludeFromRecents=true` 和 `noHistory=true`。设置页 `MainActivity` 仍然会正常出现在最近应用中，方便你继续配置。
