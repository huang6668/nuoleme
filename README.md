挪了么是一个离线、轻量、零常驻的 Android 挪车短信提醒 App。它的目标很直接：当你收到“挪车 / 移车 / 车牌命中”的短信时，不是只给你一个容易错过的普通通知，而是尽量像系统闹铃一样把你叫出来。

## 这个项目解决什么问题

很多挪车短信并不是“收到了就行”，而是需要你立刻注意到：

- 手机在锁屏、息屏或口袋里时，普通通知很容易被忽略。
- 很多人会把媒体、铃声或通知音量关掉，导致提醒不可靠。
- 常见做法往往依赖联网、常驻后台或一堆保活逻辑，体验重、权限也重。

挪了么专门只做一件事：本地匹配挪车短信，然后用更高优先级的方式提醒你。

## 它需要一直挂后台吗？

不需要。

这个 App 平时不是常驻后台服务，也不需要你一直开着前台页面。它主要依赖系统下发的 `SMS_RECEIVED` 广播，在收到新短信时被系统唤起；只有当短信命中关键词或车牌后，才会短暂进入报警流程。

从用户体验上说，它追求的是：

- 平时尽量“像没装一样”安静
- 真有挪车短信时，尽量“像闹钟一样”明显

## 核心特性

- 零联网：不申请 `INTERNET` 权限，不依赖服务端。
- 零常驻：平时不跑永久后台服务，不做轮询保活。
- 本地匹配：支持自定义关键词和车牌匹配。
- 闹铃语义提醒：优先使用系统默认闹铃铃声，不主动修改你的媒体音量。
- 全屏兜底：在锁屏、后台受限或厂商 ROM 较严格时，尽量通过高优先级 / 全屏提醒降低漏报。
- 可自测：内置“测试报警”，方便你先验证真机效果。

## 适合谁

- 停车后会留挪车联系方式，希望别错过关键短信的人
- 想要“命中后强提醒”，但不想装一个重后台、重权限 App 的人
- 在意隐私，希望短信内容只在本机处理的人

## 隐私与权限

挪了么的设计前提就是“本地处理、不过网”：

- 不联网，不上传短信
- 不集成统计、广告、崩溃上报 SDK
- 不读取历史短信
- 不需要设置成默认短信 App

运行时只申请必要权限：

- `RECEIVE_SMS`
- `POST_NOTIFICATIONS`
- `VIBRATE`
- 前台服务相关权限
- 全屏提醒相关权限

## 安装方式

目前默认通过 GitHub Actions 产出 APK：

1. 打开仓库的 [Actions](https://github.com/huang6668/nuoleme/actions) 页面。
2. 进入 `Android Build` 工作流。
3. 下载构建产物 `nuolemo-debug-apk`。
4. 安装其中的 `app-debug.apk`。

> **提示**：从 1.0.5 开始，GitHub Actions 产出的 APK 使用固定测试签名，可以直接覆盖后续 Actions 版本。此前安装的 APK 使用了随机签名，因此升级到 1.0.5 时仍需先卸载旧版一次；卸载会清除 App 设置，请提前记下自定义关键词和车牌。

## 首次使用

1. 打开 App，授予短信权限。
2. Android 13 及以上再授予通知权限。
3. 到设置页检查全屏提醒状态。
4. 先点一次“测试报警”，确认闹铃、震动和停止操作都正常。
5. 根据你的实际短信内容，补充关键词或车牌。

## 稳定性建议

Android 厂商 ROM 对后台唤起、全屏提醒和省电策略的处理差异很大。为了尽量减少漏报，建议额外检查这些系统设置：

- 允许应用自启动
- 允许后台运行或后台活动
- 电池优化改为“不受限”或“无限制”
- 不要把 App 放进一键清理名单

如果你的手机是小米、华为、OPPO、vivo、三星等定制系统，这一步通常会明显影响实际提醒成功率。

## 测试方法

- 在设置页点击“测试报警”，确认系统闹铃、震动、通知和停止按钮行为正常。
- 用另一台手机发送包含“挪车”“移车”或你自定义关键词 / 车牌的短信，验证能否触发。
- 再发送一条普通短信或验证码短信，确认不会误报。

## 开发与打包

- 仓库默认走 GitHub Actions 打包。
- 项目目标是保持离线、轻量、低打扰，不引入联网依赖和常驻后台方案。
- 如果只是修改代码或文档，优先直接改仓库并通过 GitHub 构建验证，而不是在本地额外安装一整套 Android 工具链。

## 常见问题

### 为什么它不是普通通知，而是“闹铃语义”提醒？

因为挪车短信的核心问题不是“有没有收到”，而是“会不会错过”。普通通知、媒体音量、铃声音量都可能被用户手动关掉；这个项目更希望在命中时尽量沿用系统对闹铃场景的处理方式。

### 会不会偷偷在后台一直跑？

不会。它不是一个永久常驻的后台 App。平时主要依赖系统短信广播触发，只有命中后才会进入短时报警流程。

### 需要设置成默认短信 App 吗？

不需要。它只监听系统广播里的当前短信内容，不接管短信收发。

### 为什么没有网络权限？

这是刻意保留的产品边界：短信内容只在本地处理，不上传，不联网。
